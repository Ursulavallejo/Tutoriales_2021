Displays

La propiedad display nos permite mostrar u ocultar elementos, pero también cambiar la forma en la que un elemento se muestra, como puede ser que una etiqueta de línea se comporte como un bloque y viceversa.

display : none / ocultaremos el elemento

display : inline / haremos que un elemento de bloque se comporte como un elemento de línea

display : block / haremos que un elemento de línea se comporte como un elemento de bloque

display : inline-block / se comportará como un elemento de línea en el sentido de que no provoca un salto de línea, pero aceptara algunas propiedades típicas de un bloque, como márgenes superiores e inferiores o altura

* Floats

La propiedad float es una herramienta fundamental para crear retículas. Nos permite que dos bloques se coloquen el uno al lado del otro, sin provocar saltos de línea. Sin embargo, los bloques que flotan no se llevan del todo bien con los elementos que no flotan, así que es necesario que se combinen con otras técnicas.

-Overflow : hidden

Si ponemos esta propiedad al contenedor, el contenido que lo sobrepase quedará oculto.

-Overflow : scroll

Este provocará un scroll, es decir, saldrán barras de scroll cuando el contenido sobrepase al contenedor y se podrá scrollear dentro de este contenedor.

-Overflow : auto

en principio deja al navegador elegir cuándo aplicar un scroll, pero en la práctica es similar a Overflow : scroll.

-Overflow : visible

Permite resetear al estado inicial

-Float : left

Para que un bloque se sitúe al lado de otro bloque

* Position con el navegador:

La mayor parte de las veces, que un bloque se ponga debajo de otro ya nos viene bien. En otras ocasiones, con flotar los elementos será suficiente. Pero a veces, lo que necesitamos es que un elemento salga del flujo natural de la página y se posicione de manera absoluta en alguna parte de la ventana del navegador. Para eso vamos a usar la propiedad position.

Se posicionará en la esquina superior izquierda o en el sitio se le indiquen las coordenadas en relación con el navegador.

* Position con relación a un contenedor y no con el navegador:

La solución es que el contenedor tenga "position:relative". De esta manera, un elemento posicionado absolutamente cogerá de referencia el elemento contenedor más cercano que tenga "position:relative".En este caso el Span toma como referencia el div y no el navegador.
